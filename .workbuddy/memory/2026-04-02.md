#### 1. Flash Attention（闪电般的灵力操控）

**核心问题**：传统注意力的内存瓶颈 O(S²)
- seq_len=4096, batch=32 时，单层需要 ~4GB 显存
- 32 层模型需要 128GB 显存，远超大多数 GPU 容量

**核心技术**：
- **分块计算（Tiling）**：将序列分成小块逐块处理
- **融合核（Fused Kernel）**：将 softmax 和 matmul 融合在一个 CUDA 核中
- **在线 Softmax（Online Softmax）**：逐步累积 softmax 结果，只需存储两个标量（max_val 和 sum_exp）
- **重计算（Recomputation）**：前向传播不保存中间结果，反向传播时重新计算（用算力换内存）

**内存优化**：O(S²) → O(S×d_k)，节省 20 倍显存

**完整实现**：包含简化版 Flash Attention 算法和 PyTorch 2.0+ 的 `F.scaled_dot_product_attention` 使用示例

#### 2. RoPE（旋转位置编码）— 相对位置感知

**核心问题**：正弦位置编码是绝对的，无法表达相对位置关系

**核心技术**：
- **复数旋转**：用复数旋转编码相对位置信息
  ```
  v_rotated = v × e^(iθ) = [x·cosθ - y·sinθ, x·sinθ + y·cosθ]
  ```
- **群性质**：Rot(θ₁) × Rot(θ₂) = Rot(θ₁ + θ₂)
  - 相对距离通过简单的角度减法得到
- **高维扩展**：将 d_model 维向量分成 d_model/2 个二维对，每个对独立旋转
- **预计算频率**：`freqs_cis = torch.polar(torch.ones_like(freqs), freqs)`

**优势**：
- **外推能力强**：可以自然外推到训练时未见过的序列长度
- **零参数**：完全解析式，无需训练
- **相对位置**：天然支持相对位置关系

**完整实现**：包含 `precompute_freqs_cis`、`apply_rotary_emb`、`RoPEMultiHeadAttention`

#### 3. GQA（分组查询注意力）— 计算效率的极致

**核心问题**：推理时 KV Cache 内存占用巨大

**核心技术**：
- **MHA（多头注意力）**：32 个 Q、K、V 头，KV Cache = 32×
- **MQA（多查询注意力）**：32 个 Q 头共享 1 个 K/V，KV Cache = 1×
- **GQA（分组查询注意力）**：32 个 Q 头分成 8 组，每组 4 个 Q 共享 1 个 K/V
  ```
  groups = num_heads / num_kv_heads = 32 / 8 = 4
  ```

**性能对比**：
| 架构 | Q 头数 | K/V 头数 | KV Cache 大小 | 表达能力 |
|------|--------|-----------|---------------|----------|
| MHA | 32 | 32 | 32× | 完整（最强） |
| GQA (8 groups) | 32 | 8 | 0.25× | 优秀（95% 性能） |
| MQA | 32 | 1 | 0.03× | 较弱（90% 性能） |

**内存优化**：KV Cache 减少 4 倍（GQA）到 32 倍（MQA）

**完整实现**：`GroupedQueryAttention` 类，包含 `repeat_interleave` 广播逻辑

#### 4. LoRA（低秩自适应）— 轻量级微调术

**核心问题**：全量微调需要巨大资源（7B 模型需要 40GB 显存）

**核心技术**：
- **低秩分解**：ΔW = B @ A，其中 B ∈ R^(d_in × r), A ∈ R^(r × d_out)
- **参数效率**：r=16 时，可训练参数仅 0.39%（减少 250 倍）
- **初始化策略**：A 使用 Kaiming 初始化，B 初始化为零（初始时增量为零）
- **缩放因子**：`scaling = alpha / rank`，用于平衡增量的贡献

**QLoRA 结合**：
- 4-bit 量化：模型大小 14GB → 3.5GB
- 训练显存：40GB → 12GB（在 RTX 3090 24GB 上成为可能）

**完整实现**：
- `LoRALinear` 类：完整的 LoRA 线性层
- `inject_lora_to_model` 函数：自动注入到现有模型
- 使用示例：加载 LLaMA、注入 LoRA、冻结非 LoRA 参数
