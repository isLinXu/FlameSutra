**3. 动态量化 vs 静态量化**
- 动态量化：权重量化、激活推理时量化（适合 RNN/NLP）
- 静态量化：权重和激活都量化（适合 CNN，速度更快）

#### B.3 知识蒸馏（Knowledge Distillation）

**1. 软标签 vs 硬标签**
- 硬标签：[0, 0, 1, 0]
- 软标签：[0.01, 0.05, 0.92, 0.02] - 包含类间相似性

**2. 温度参数（Temperature）**
- 公式：p_i = exp(z_i / T) / Σ_j exp(z_j / T)
- T=1：正常 softmax
- T>1：分布更平滑（信息更丰富）
- 完整的 `softmax_with_temperature` 实现

**3. 蒸馏损失函数**
- 完整的 `DistillationLoss` 类实现
- L_distill = α * L_soft + (1-α) * L_hard
- α=0.7，temperature=5.0

**4. 完整蒸馏训练**

#### 7.1 为何需要生成式模型？

#### 7.3 扩散模型（Diffusion Models）

**1. 前向扩散（加噪）**
- 过程：x_0 → x_1 → ... → x_T（纯噪声）
- 公式：x_t = sqrt(1 - β_t) * x_{t-1} + sqrt(β_t) * ε
- β_t：加噪调度（schedule）
- ε：标准高斯噪声

**2. 反向扩散（去噪）**
- 过程：x_T → x_{T-1} → ... → x_0（干净图像）
- 公式：x_{t-1} = (x_t - sqrt(1 - α_t) / sqrt(α_t) * ε_θ(x_t, t)) / sqrt(β_t)
- ε_θ(x_t, t)：神经网络预测的噪声

**3. DDPM 完整实现**
- `GaussianDiffusion` 类：
  - 前向扩散：`q_sample`
  - 反向扩散：`p_sample`
  - β/α/ᾱ 调度计算
- `UNet` 类：去噪网络
  - Encoder（下采样）
  - Bottleneck
  - Decoder（上采样）
  - Skip connections
  - 时间嵌入
- `train_ddpm`：训练函数
- `generate_image`：生成函数（从纯噪声开始）

**4. DDPM 生成效果**
- 从纯噪声 (T=1000)：████████████████████████
- 去噪 500 步 (T=500)：▓▓▓▓▓▓▓░░░░░░░░░░
- 去噪 100 步 (T=100)：▓▓▓▓░░░░░░░░░░░░░
- 最终图像 (T=0)：一只清晰的猫！

#### 7.4 变分自编码器（VAE）

**1. VAE 完整实现**
- `VAE` 类：
  - Encoder：编码器（x → μ, σ）
  - `reparameterize`：重参数化技巧（z = μ + σ × ε）
  - Decoder：解码器（z → x̂）
- Loss = Reconstruction Loss + KL Divergence
- `vae_loss_function`：VAE 损失函数
  - BCE（二值交叉熵）：重建损失
  - KLD（KL 散度）：KL(N(μ, σ²) || N(0, I)) = -0.5 × Σ(1 + log(σ²) - μ² - σ²)
- `train_vae`：训练函数
- `generate_from_vae`：从潜在空间生成

**2. 潜在空间可视化**
- 每个数字占据潜在空间的一个区域
- 从两个数字之间采样，会得到中间状态
- 可以通过插值实现平滑过渡

#### 7.5 生成对抗网络（GAN）

**1. GAN 基础架构**
- 生成器（Generator）：随机噪声 z → 生成假图像 x̂
- 判别器（Discriminator）：真图像 x → 判别是否为真
- 目标：
  - 生成器：最大化 D(G(z))（欺骗判别器）
  - 判别器：最大化 D(x) - D(G(z))（正确判断）

**2. DCGAN 完整实现**
- `Generator` 类：DCGAN 生成器
  - 转置卷积（上采样）
  - BatchNorm、ReLU
  - 输出：Tanh
- `Discriminator` 类：DCGAN 判别器
  - 标准卷积
  - BatchNorm、LeakyReLU
  - 输出：Sigmoid（概率）
- `train_gan`：训练函数
  - 训练判别器：loss_D = loss_real + loss_fake
  - 训练生成器：loss_G（希望判别器认为假图像是真的）
  - 梯度更新

**3. GAN 训练稳定性技巧**
- 标签平滑：使用 0.9/0.1 代替 1/0（避免判别器过于自信）
- WGAN-GP：使用 Wasserstein 距离（梯度更稳定）
- 谱归一化：限制判别器的 Lipschitz 常数（防止梯度爆炸）
- Two-Time-Scale Update (TTUR)：生成器和判别器使用不同学习率（平衡训练速度）

### 一、附录 A 扩充：高级训练技巧（约 1,300 行）

#### A.3 高级训练技巧 — 突破瓶颈的秘术

**1. 学习率调度 — 温火慢炖的艺术**

- Warmup（预热）调度器：
  - `WarmupScheduler` 类：前期学习率从小到大
  - 避免模型初期震荡
  - 公式：lr = base_lr * min(step / warmup_steps, 1.0)

- Cosine 退火调度：
  - `cosine_annealing_with_warmup` 函数
  - 平滑的学习率变化
  - 后期逐渐降低，有助于收敛到更优解
  - 公式：lr = min_lr + 0.5 * (base_lr - min_lr) * (1 + cos(π * progress))

**2. 梯度裁剪 — 避免走火入魔**

- 梯度爆炸检测：
  - `check_gradient_explosion` 函数
  - 监控最大梯度范数、平均梯度范数
  - 统计爆炸参数数量

- 梯度裁剪策略：
  - 按范数裁剪：`clip_grad_norm_`（推荐）
  - 按值裁剪：`clip_grad_value_`
  - 保持梯度方向不变，只是缩放梯度大小

- 自适应梯度裁剪（AGC）：
  - `AdaptiveGradientClipping` 类
  - 每个参数的裁剪阈值 = λ * ||W||
  - 大权重的参数允许更大的梯度
  - 论文：AGC: Adaptive Gradient Clipping (https://arxiv.org/abs/2102.06171)

**3. 混合精度训练 — 节省灵力的法门**

- FP16 训练的优势：
  - 显存占用减半（FP16: 2 bytes, FP32: 4 bytes）
  - 计算速度提升（Tensor Core 专门优化 FP16）
  - 更大的 batch size

- 原生 AMP（Automatic Mixed Precision）训练：
  - `train_amp` 函数
  - `autocast` 上下文：自动选择 FP16/FP32
  - `GradScaler`：梯度缩放，防止下溢出

- 梯度缩放原理：
  - `ManualGradScaler` 类（理解原理）
  - FP16 范围小，小梯度可能下溢为 0
  - 解决：将梯度乘以 2^16，反向传播后除回来
  - 策略：没有 inf/nan 逐步增大，出现 inf/nan 缩小

**4. 标签平滑 — 避免过度自信**

- 为什么需要标签平滑：
  - 传统 one-hot 标签让模型过度自信
  - 模型可能记住训练集噪声
  - 泛化性能下降

- 标签平滑实现：
  - `label_smoothing_loss` 函数
  - one-hot: [0, 0, 1, 0, 0]
  - 平滑后: [0.025, 0.025, 0.9, 0.025, 0.025]
  - 公式：y_smooth = (1 - ε) * y_one_hot + ε / K

- PyTorch 内置支持：
  - `nn.CrossEntropyLoss(label_smoothing=0.1)`（PyTorch 1.10+）

**5. 梯度累积 — 突破显存限制**

- 为什么需要梯度累积：
  - 小显存只能跑 batch_size=4
  - 但大 batch size（如 16）收敛更快

- 梯度累积实现：
  - `train_with_gradient_accumulation` 函数
  - 分 4 个 mini-batch，每个 batch_size=4
  - 累积 4 次梯度后，再更新参数
  - 相当于 batch_size=16 的效果

- 梯度累积 + AMP 组合：
  - `train_amp_with_accumulation` 函数
  - 结合两者的优势
  - 既节省显存，又提升速度

**6. Early Stopping — 见好就收的智慧**

- 早停机制：
  - `EarlyStopping` 类
  - 监控验证集指标
  - 连续 patience 个 epoch 没有提升，停止训练
  - 参数：patience=7, min_delta=0.0, mode='min'

- 完整早停训练：
  - `train_with_early_stopping` 函数
  - 训练 + 验证循环
  - 保存最佳模型
  - 加载最佳模型参数

**7. 数据增强 — 扩充修炼资源**

- 图像数据增强：
  - `get_image_augmentation` 函数
  - 随机翻转、旋转、裁剪、颜色抖动、标准化
  - 训练集和验证集不同的增强策略

- 文本数据增强：
  - `text_augmentation` 函数
  - 同义词替换、随机插入、随机删除、回译
  - 使用 nlpaug 库

- Mixup 和 CutMix 增强技术：
  - `mixup_data` 函数：将两张图片线性插值
  - `cutmix_data` 函数：剪切混合
  - `mixup_criterion` 函数：Mixup 损失函数
  - 公式：x_mix = λ * x_1 + (1-λ) * x_2

**8. 权重初始化 — 良好的开端**

- 权重初始化策略：
  - `init_weights` 函数
  - Xavier/Glorot 初始化：适用于 tanh, sigmoid
  - Kaiming/He 初始化：适用于 ReLU, LeakyReLU
  - BatchNorm 初始化

- 预训练模型初始化：
  - `init_from_pretrained` 函数
  - 从预训练模型初始化
  - 加载匹配的权重
  - 随机初始化剩余层

- 层级学习率衰减：
  - `get_layerwise_lr` 函数
  - 浅层（靠近输入）使用较小的学习率
  - 深层（靠近输出）使用较大的学习率
  - 公式：lr_layer = base_lr * decay^(depth - 1)

#### D.2 图像分割修炼 — 语义分割

**1. 模型定义**

- U-Net 模型：
  - `UNet` 类
  - 编码器：逐步下采样，提取特征
  - 解码器：逐步上采样，恢复空间分辨率
  - 跳跃连接：将编码器的特征传到解码器

- U-Net 组件：
  - `DoubleConv`：双重卷积块（Conv → BN → ReLU → Conv → BN → ReLU）
  - `Down`：下采样模块（MaxPool → DoubleConv）
  - `Up`：上采样模块（转置卷积 → Conv，带跳跃连接）

**2. 损失函数**

- Dice Loss：
  - `DiceLoss` 类
  - 公式：Dice = 2 * |A ∩ B| / (|A| + |B|)
  - Loss = 1 - Dice
  - 优点：对类别不平衡不敏感，直接优化 IoU

- 组合损失：
  - `CombinedLoss` 类
  - CrossEntropy + Dice
  - 公式：Loss = α * CE + (1-α) * Dice
  - 结合分类和分割的优化目标

**3. 评估指标**

- IoU（交并比）：
  - `calculate_iou` 函数
  - 公式：IoU = |预测 ∩ 真实| / |预测 ∪ 真实|
  - 每个类别单独计算，然后平均

- 像素准确率：
  - `calculate_pixel_accuracy` 函数
  - 公式：正确像素数 / 总像素数

- 分割模型评估：
  - `evaluate_segmentation` 函数
  - 计算：loss、mIoU（平均 IoU）、pixel_accuracy

**4. 数据集**

- 语义分割数据集：
  - `SegmentationDataset` 类
  - 图像和标签分别加载
  - 支持不同的图像/标签变换

- 分割数据增强：
  - `get_segmentation_augmentation` 函数
  - 图像：颜色抖动、归一化
  - 标签：只做几何变换，不归一化

#### D.3 序列生成修炼 — 简易语言模型

**1. 模型定义**

- LSTM 语言模型：
  - `LSTMLanguageModel` 类
  - 输入：文本序列
  - 输出：下一个词的概率分布
  - 架构：Embedding → LSTM → FC

**2. 文本生成**

- 采样下一个 token：
  - `sample_next_token` 函数
  - Temperature：控制随机性（高 → 多样性，低 → 确定性）
  - Top-k：只从概率最高的 k 个候选中采样
  - Top-p（Nucleus）：从累积概率达到 p 的候选中采样

- 生成文本：
  - `generate_text` 函数
  - 自回归生成
  - 支持温度、Top-k、Top-p 采样
  - 遇到结束符则停止

- 束搜索解码：
  - `beam_search_decode` 函数
  - 同时维护 top-k 个候选序列
  - 每一步扩展所有候选
  - 保留分数最高的 k 个序列
  - 优点：比贪婪搜索更好，比纯随机搜索更稳定

**3. 评估指标**

- 困惑度（Perplexity）：
  - `calculate_perplexity` 函数
  - 公式：PPL = exp(average_loss)
  - 含义：模型对测试集的"困惑程度"
  - 越低越好，PPL=10 表示模型在 10 个等概率的选项中犹豫

- BLEU 分数：
  - `compute_bleu` 函数
  - 计算参考文本和生成文本的相似度
  - 使用 n-gram 匹配
  - 几何平均 + 短句惩罚
